# Diffusion-DICE 2D Bandit Toy Case Demo

This directory contains a self-contained demonstration of the **2D Bandit Toy Case** from the NeurIPS 2024 paper **"Diffusion-DICE: In-Sample Diffusion Guidance for Offline Reinforcement Learning"** (Mao et al., 2024), presented in the ICML 2025 tutorial *"Generative AI Meets Reinforcement Learning"*.

The purpose of this toy case is to demonstrate the phenomenon of **value-function error exploitation** in offline reinforcement learning and how Diffusion-DICE successfully alleviates it using **In-Sample Guidance Learning (IGL)** compared to other state-of-the-art diffusion policy paradigms.

---

## Mathematical Background & Problem Setup

### 1. Annular Dataset (Data Support)
The offline dataset represents limited data coverage. The actions $a \in \mathbb{R}^2$ follow a bivariate standard normal distribution restricted to an **annular (ring-shaped)** region:
$$1.5 \leq \|a\|_2 \leq 2.5$$
The area inside the ring ($\|a\|_2 < 1.5$) represents the **out-of-distribution (OOD)** region, where no training data exists.

### 2. Reward & Critic Overestimation
*   **Ground Truth Reward $R(a)$**: The true reward peaks on the outer circle of the data support ($\|a\|_2 = 2.5$) and is symmetric along the horizontal axis ($\cos(2\theta)$). It is 0 at the origin $(0, 0)$.
*   **Learned Critic $\hat{R}(a)$**: Trained on the offline dataset $\mathcal{D}$. Due to the lack of training data in the center OOD region, the function extrapolates. In offline RL, this leads to **overestimation errors** (e.g., predicting an erroneously high reward at the center).

### 3. Comparison of Algorithms
*   **IDQL (Select-only)**: Generates candidate actions from the behavior diffusion model $\hat{\pi}_b(a)$ (trained to mimic the dataset). Since diffusion models can make small fitting errors, some candidates may fall slightly inside the inner ring. IDQL selects the action from these candidates that maximizes the learned critic $\hat{R}(a)$. Because of overestimation in the center, IDQL exploits this error and selects OOD, sub-optimal actions in the center.
*   **QGPO (Guide-only)**: Guides the reverse diffusion process using the gradient of the learned critic:
    $$\epsilon_{guided} = \epsilon_b - \eta \sqrt{1 - \bar{\alpha}_t} \nabla_{a_t} \hat{R}(a_t)$$
    Since $\hat{R}$ peaks at the center, the gradient points inward, pulling the generated actions directly into the OOD center.
*   **Diffusion-DICE (Guide-then-select)**: Learns in-sample weights $w^*(a) = (f')^{-1}(\frac{\hat{R}(a) - V}{\alpha})$ using Fenchel-Rockafellar duality and a piecewise $f$-divergence. It trains a guidance network $g_\theta(a_t, t)$ using only in-sample data. Sampling is guided by $\nabla_{a_t} g_\theta(a_t, t)$, which pushes actions towards high-value *in-distribution* regions (the outer peaks). The select step then chooses the best among these in-distribution candidates, avoiding OOD overestimation.

## File Structure

- `toycase_bandit.py`: The self-contained training and evaluation script.
- `requirements.txt`: Python package requirements.
- `toycase_results.png`: Output figure showing the visual comparison (generated after running).

---

## Installation and Running

### Prerequisites
Make sure you have Python (version >= 3.8) installed.

### Step 1: Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 2: Run the Demo
```bash
python toycase_bandit.py
```

---

## Expected Results

After running the script, `toycase_results.png` will be generated in this folder. It contains six subplots:
1.  **Dataset Actions $\pi^{\mathcal{D}}$**: Scatter plot showing the annular shape of the training dataset.
2.  **Ground Truth Reward $R$**: Contour plot showing the two true reward peaks on the outer ring.
3.  **Learned Reward $\hat{R}$**: Contour plot of the learned critic, highlighting the high overestimation peak in the center OOD region.
4.  **QGPO (Guide-only)**: Scatter plot of actions generated by QGPO, which are all clustered at the origin $(0, 0)$ due to gradient-based error exploitation.
5.  **IDQL (Select-only)**: Scatter plot of actions selected by IDQL, showing that it also gets pulled into the center due to selecting from unguided candidates.
6.  **Diffusion-DICE (Ours)**: Scatter plot showing actions successfully generated on the true outer peaks, completely avoiding the center OOD region.
